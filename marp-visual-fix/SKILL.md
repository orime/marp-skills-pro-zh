# Marp Skills Pro - 视觉排版自愈助手 (Visual Fix)

这不仅仅是一套故障修复技能，更是“糟糕版式”转为优质展示的重要桥梁助手。

## 触发定位
- “帮我修一下文字撑破页面出框的问题”
- “某一张幻灯片的图片太巨大了，能不能变小点并且有排版”
- “导出后的 PDF 里字体挤成了死胡同”
- “这页代码块挤到底部被切断了怎么办”
- “帮我对第三页的排版做高光改版调整”

## 执行工作流

1. **破绽定位**：定位发生外溢、破损、或者视觉结构过分劣质（如图片生硬贴墙）的幻灯片。
2. **致病分析**：审视内容量，推论是否超负荷、Markdown组件未结束或主题 CSS 与内容不匹配。
3. **多维化修补**：
   - 基础解法：缩写超长的句子，将文字量分配到后续的多张幻灯片上。
   - 破界解法：通过自定义 CSS 内部网格标签（如将其放置进并列式 `div.grid`），通过布局控制强行锁死高度溢出。
4. **方案展示与应用**：完成代码替换及排版重建。

## 链接知识点
- **marp-knowledge**：参考指令调整图片比例，寻找合适的内部背景类来控制布局。

## “望闻问切” 黄金诊断矩阵
1. **文字井喷与出血溢出**
   - 单页是否含过多要点？→ 拆页处理（分页指令）。
   - 副标题尺寸过大？ → 将冗余转变成小字体的注脚或正文段落。
   - *（高级）*是否能改用双栏网格 `grid` 平衡重量感？
2. **配图灾难（尺寸过载或留白失调）**
   - 插入原大尺寸图是否顶飞了标题？ → 换用 `![bg right:40% fit](...)` 指令融合背景，或采用“玻璃态图像框”包裹图元素保证圆角及投影质感。
3. **框架破碎**
   - 是否使用了违规标签？或表格占据了惊人的宽度？ → 控制宽度比例，对超长 URL 或连续英文字符施加换行方案。
4. **导出时渲染失效**
   - 系统中缺省的现代字体系列是否存在跨平台异常？建议增加兜底字体。
